10-Step Blue/Green Deployment Workflow

1. Setup GitLab Infrastructure - Deploy GitLab server and CI/CD environment
2. Create Base Infrastructure - Deploy VPC, ALB, and target groups
3. Create Blue Cluster - Deploy initial production cluster (Version N)
4. Setup Next Version Branch - Prepare GitOps branches for new version
5. Create Green Cluster - Deploy new cluster with Version N+1
6. Enable Internal Testing - Route test traffic to green cluster
7. Promote Green Cluster - Switch production traffic to green
8. Rollback (if needed) - Quick revert to blue cluster
9. Merge Version Branches - Update GitOps repositories
10. Delete Old Cluster - Clean up previous version resources
